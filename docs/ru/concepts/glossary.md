# Полный глоссарий основных терминов объектного хранилища (100 терминов)

| № | Термин | Английский | Описание |
|------|--------------------------|------------------------------|--------------------------------------------------------------------------|
| 1 | Объектное хранилище | Object Storage | Архитектура хранения данных в виде объектов (Object), заменяющая традиционную иерархическую файловую структуру |
| 2 | Корзина | Bucket | Контейнер для хранения объектов с глобально уникальным пространством имен |
| 3 | Объект | Object | Базовая единица хранения, содержащая данные, метаданные и уникальный идентификатор (Object Key) |
| 4 | Метаданные | Metadata | Информация в виде пар ключ-значение, описывающая свойства объекта (например, тип файла, время создания) |
| 5 | S3-совместимость | S3-Compatible | Служба хранения, совместимая со стандартом API Amazon S3 |
| 6 | Долговечность данных | Data Durability | Вероятность долговременного сохранения данных в системе без потерь (например, 99.999999999%) |
| 7 | Репликация | Replication | Технология избыточности, обеспечивающая безопасность данных через создание нескольких копий |
| 8 | Кодирование стирания | Erasure Coding | Технология разделения и кодированного хранения данных для достижения высокой надежности с меньшим объемом пространства |
| 9 | Холодное хранилище | Cold Storage | Тип хранения с низкой стоимостью для редко используемых данных (например, архивные данные) |
| 10 | Управление жизненным циклом | Lifecycle Management | Политика автоматического перемещения/удаления объектов (например, перемещение в холодное хранилище через 30 дней) |
| 11 | Версионирование | Versioning | Сохранение исторических версий объектов для предотвращения перезаписи |
| 12 | Класс хранения | Storage Class | Уровни хранения с различной производительностью/стоимостью (стандартный, низкочастотный, архивный) |
| 13 | Ключ доступа | Access Key | Ключ аутентификации для API-запросов (Access Key ID + Secret Access Key) |
| 14 | Регион | Region | Географическое расположение инфраструктуры хранения (например, Восток Китая 1, Запад США) |
| 15 | Зона доступности | Availability Zone (AZ) | Изолированные дата-центры с независимым питанием/сетью в пределах одного региона |
| 16 | Конечная точка | Endpoint | Доменное имя для доступа к службе хранения (например, us-east1.rustfs.com) |
| 17 | RESTful API | RESTful API | Спецификация проектирования API на основе протокола HTTP |
| 18 | Многочастная загрузка | Multipart Upload | Механизм разделения больших файлов на части для загрузки с последующим слиянием |
| 19 | Предподписанный URL | Pre-Signed URL | Временная ссылка доступа с ограниченным сроком действия |
| 20 | Серверное шифрование | SSE | Автоматическое шифрование данных на стороне сервера (SSE-S3/SSE-KMS/SSE-C) |
| 21 | Клиентское шифрование | CSE | Шифрование на стороне клиента перед загрузкой |
| 22 | Кросс-региональная репликация | Cross-Region Replication | Автоматическая репликация объектов между географическими регионами |
| 23 | Список контроля доступа | ACL | Список правил, контролирующих права доступа к корзинам/объектам |
| 24 | Политика корзины | Bucket Policy | Политика детального контроля прав доступа на основе JSON |
| 25 | IAM | Identity and Access Management | Система централизованного управления правами доступа пользователей/ролей |
| 26 | Уведомления о событиях | Event Notification | Отправка уведомлений в очереди сообщений/вычислительные функции при возникновении событий |
| 27 | Озеро данных | Data Lake | Хранилище для централизованного хранения структурированных/неструктурированных данных |
| 28 | Соответствие требованиям | Compliance | Соответствие требованиям законодательства о хранении данных, таким как GDPR, HIPAA |
| 29 | Журналирование и аудит | Logging & Audit | Запись всех журналов операций API для аудита |
| 30 | Мониторинг и оповещения | Monitoring & Alerting | Мониторинг использования хранилища/количества запросов в реальном времени с триггерами оповещений |
